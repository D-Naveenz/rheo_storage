# Runbooks

Use this folder for repeatable operational guidance that is too detailed for `AGENTS.md`
or `project-guidelines.md`.

## What Belongs Here
- Release checklists and operator flows.
- CI and package verification procedures.
- Repo tooling workflows that require multiple steps or environment notes.

## What Does Not Belong Here
- Durable repository guardrails that should apply to every session.
- Codex app setup and action wiring that belongs in `.codex/`.

Add new runbooks here as the repo accumulates more multi-step operational knowledge.
